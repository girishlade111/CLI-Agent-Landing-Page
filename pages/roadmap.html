LLM Scaling Roadmap

Four clear stages of parameter scaling, each unlocking new capabilities.

Stage 1

Initial

30B

Initial coding model, good for single-GPU deployments.

Purpose: Fast iteration, quick code generation.
Trade-off: Limited context (~32K). Best for single-file tasks.
Stage 2

Enhanced

180B

Deeper reasoning for architecture and security reviews.

Purpose: Multi-file context, complex refactoring.
Trade-off: Higher latency (2-3x vs Stage 1). Requires multi-GPU.
Stage 3

Primary

250B

Primary backbone for IDE and multi-agent workflows.

Purpose: Agent coordination, large codebase reasoning.
Trade-off: Significant compute. Best for high-value complex tasks.
Stage 4

Frontier

1T

Frontier-scale model for complex, multi-system enterprises.

Purpose: System-level reasoning, enterprise-wide coordination.
Trade-off: Highest compute requirements. Reserved for dedicated infrastructure.
